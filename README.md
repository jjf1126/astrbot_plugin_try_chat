# AstrBot 插件：主动聊天触发器 (try_chat)

## 插件简介

`astrbot_plugin_try_chat` 是一款为 AstrBot 设计的智能化主动聊天控制插件。该插件打破了传统机器人只能“被动回复”的限制，赋予了 AI 基于时间、聊天上下文和概率分布主动发起对话的能力。

通过定时任务与泊松分布（Poisson distribution）算法，插件会周期性地“掷骰子”来决定是否唤醒 AI。唤醒后，插件会结合当前会话的专属人格、最近聊天记录以及当前北京时间，使用自定义的决策大模型和提示词来最终判断是否向用户发送消息。这种机制能够极大地提升 AI 伴侣的拟真度，使其表现得像一个真实的人类朋友，会在恰当的时机主动找你聊天。

## 功能说明

*   **定时与概率双重控制**：默认每半小时对已开启的私聊会话进行一次轮询检测，使用泊松分布计算触发概率。
*   **概率动态补偿**：如果本次“掷骰子”未命中，或者大模型判断当前不适合发消息（Skip），插件会自动增加下一次触发的命中概率，确保主动聊天行为在合理的时间范围内必然发生。
*   **多会话独立管理**：支持同时配置多个私聊会话（指定机器人账户与用户 QQ/会话 ID），每个会话独立计算概率，并应用其专属的人格配置。
*   **智能上下文感知**：在决定是否发消息时，会自动提取该会话的最近聊天记录和当前北京时间。
*   **高度自定义提示词**：支持自定义“决策提示词”和“触发提示词”，内置 `{{current time}}` 变量，让 AI 具备准确的时间观念。
*   **独立决策模型**：允许为“判断是否发消息”这一步配置独立的大语言模型（LLM），与日常聊天的模型解耦，优化成本与效果。

## 插件流程

本插件的内部工作流程及核心逻辑如下：

1.  **定时轮询 (Scheduled Polling)**：
    *   插件内部注册了一个周期为 30 分钟的定时任务。
    *   任务触发时，遍历配置中所有已启用的 `目标会话 (Session)`。
2.  **概率判定 (Probability Check)**：
    *   针对每个会话，基于泊松分布算法生成一个随机数，与当前该会话的“阈值概率”进行比对。
    *   **未命中**：跳过本次判定，并将该会话的下次触发概率按设定步长增加。
    *   **命中**：进入下一步 AI 决策流程。
3.  **上下文组装 (Context Assembly)**：
    *   获取目标会话的 `Unified Message Origin (UMO)`。
    *   调用 AstrBot 核心接口获取该会话绑定的 **人格设定 (Persona)** 和 **最近聊天记录 (History)**。
    *   获取当前系统的北京时间，并将其替换到决策提示词中的 `{{current time}}` 变量。
4.  **AI 智能决策 (AI Decision)**：
    *   将组装好的系统提示词（包含人格、历史记录、时间、决策要求）发送给配置的 **决策模型**。
    *   模型根据提示词要求输出结果（例如输出 `YES` 或 `NO`）。
5.  **动作执行 (Execution)**：
    *   **决策为 NO（不发送）**：视为放弃本次主动聊天，结束流程，并增加下次“掷骰子”的命中概率。
    *   **决策为 YES（发送）**：将用户配置的“触发提示词”（如：“现在请主动向我打个招呼”）作为一条虚拟的用户消息，加入 AstrBot 的对应会话消息队列中。
    *   AstrBot 接收到该虚拟消息后，会使用该会话日常聊天的模型和人格生成最终的回复内容，并主动发送给用户。发送成功后，重置该会话的触发概率为初始值。

## 使用方法

### 1. 指令说明

插件提供了以下指令供管理员或用户在聊天界面直接交互：

*   `/try_chat_status`
    *   **功能**：查看当前所有开启自动聊天会话的状态与下次触发的概率信息。
    *   **示例输出**：`会话 [QQ:12345678] - 当前概率: 15% - 状态: 运行中 - 上次触发: 2小时前`
*   `/try_chat_force`
    *   **功能**：强制对当前所在的会话进行一次主动聊天判定与触发。
    *   **使用场景**：用于测试配置是否正确，或者在不想等待半小时定时器的情况下立即让 AI 走一遍决策流程。注意，此指令依然会经过“AI 智能决策”步骤，如果 AI 认为当前不适合聊天，依然可能不会发送消息。

### 2. 提示词变量使用

在 WebUI 配置面板中填写“决策提示词”时，务必合理使用变量以增强 AI 的判断能力：
*   `{{current time}}`：该变量会在每次检测时被自动替换为当前的北京时间（例如：`2026年5月10日 16:30:00`）。

**决策提示词编写建议**：
> "你是一个贴心的朋友。请阅读以下最近的聊天记录，并结合当前时间 {{current time}}，判断现在是否适合主动找我聊天。如果在深夜且之前已经互道晚安，请回复 NO。如果距离上次聊天已经过了很久，且现在是白天，请回复 YES。只能回复 YES 或 NO。"

## 配置说明

在 AstrBot 的 WebUI 管理面板中，找到 `astrbot_plugin_try_chat` 插件的设置项，可进行以下配置：

| 配置项 (Schema Key) | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `enabled_sessions` | `list` | `[]` | 启用的会话列表。请填写需要开启此功能的会话标识（通常为 `平台标识-机器人ID-用户ID`）。 |
| `decision_model_id` | `string` | `""` | 用于执行“是否聊天”决策的大模型 ID。留空则默认使用 AstrBot 全局默认模型。可以通过 AstrBot 提供的提供商选择器进行选择。 |
| `decision_prompt` | `text` | 见上文建议 | 决策提示词。告诉 AI 如何根据时间、记录判断是否发消息。**必须包含**判断标准，支持 `{{current time}}` 变量。 |
| `trigger_prompt` | `string` | `"现在请根据当前时间和我们的关系，主动向我发起一次聊天。"` | 当决策模型判定需要发消息时，压入消息队列的触发提示词。日常聊天模型将根据此提示词生成最终的主动回复。 |
| `base_probability` | `float` | `0.05` | 基础触发概率（0~1之间）。例如 0.05 代表初始 5% 的命中率。 |
| `probability_step` | `float` | `0.02` | 概率递增步长。每次未命中或 AI 拒绝发送时，下次概率将增加此数值。 |

## 注意事项

1.  **时区设置**：插件内部默认获取的时间为北京时间（Asia/Shanghai），请确保宿主机时间准确，以免传递给大模型的时间产生偏差。
2.  **消息队列注入**：当插件决定发起聊天时，是将“触发提示词”模拟为用户发送的消息送入 AstrBot 核心处理流程。这意味着最终生成消息消耗的 Token 计费归属于该会话日常使用的对话模型。
3.  **隐私保护声明**：本插件读取的“最近聊天记录”仅用于在本地内存中构建发给大模型的上下文提示词，绝不会进行任何未经授权的云端上传、存储或滥用。请确保您配置的 LLM 服务商符合您的隐私安全要求。
4.  **模型能力要求**：由于“决策”步骤需要模型具备良好的逻辑判断能力和指令遵循能力（如严格输出 YES/NO），建议 `decision_model_id` 配置为推理能力较强的模型。
5.  **频率限制**：请勿将 `base_probability` 和 `probability_step` 设置过高，以免造成 AI 频繁主动发消息，对用户造成打扰（建议保持默认值或根据实际体验微调）。
