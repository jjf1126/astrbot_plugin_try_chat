# AstrBot 插件：主动聊天触发器 (try_chat)

## 插件简介

`astrbot_plugin_try_chat` 是一款为 AstrBot 设计的智能化主动聊天控制插件。该插件打破了传统机器人只能“被动回复”的限制，赋予了 AI 基于时间、聊天上下文、长期记忆和概率分布主动发起对话的能力。

通过定时任务算法，插件会周期性地“掷骰子”来决定是否唤醒 AI。唤醒后，插件会结合当前会话的专属人格、最近聊天记录、长期记忆（需配合 LivingMemory 插件）以及当前北京时间，使用自定义的决策大模型和提示词来最终判断是否向用户发送消息。这种机制能够极大地提升 AI 伴侣的拟真度，使其表现得像一个真实的人类朋友，会在恰当的时机主动找你聊天。

## 功能说明

*   **定时与概率双重控制**：支持自定义轮询间隔（默认半小时）及随机波动时间，对已开启的会话进行轮询检测。
*   **概率动态补偿与重置**：
    *   如果本次“掷骰子”未命中，或者大模型判断当前不适合发消息，插件会自动增加下一次触发的命中概率。
    *   **当用户主动发送消息时**，该会话的触发概率会立即重置为基础概率，避免在用户活跃时过度打扰。
*   **多会话独立管理**：支持同时配置多个会话，每个会话的计时器、命中概率及触发状态完全独立计算，互不干扰。
*   **智能上下文感知**：在决定是否发消息时，会自动提取该会话的最近聊天记录（长度可配置）和当前北京时间。
*   **LivingMemory 长期记忆联动**：如果系统中安装了 LivingMemory 插件，本插件在判定和触发阶段会自动调用记忆搜索（支持自定义搜索关键词，如“最近的约定”），让 AI 的主动发话更具连贯性和人情味。
*   **大模型接口容错重试**：针对大模型接口偶发的 503/504 等繁忙报错，内置了自动重试机制（重试次数可配置），提高主动消息的送达成功率。
*   **历史记录无缝衔接**：主动触发的系统提示词和 AI 的回复会自动无缝保存到当前会话的上下文中。
*   **独立决策模型**：允许为“判断是否发消息”这一步配置独立的大语言模型（LLM），与日常聊天的模型解耦，优化成本与效果。

## 插件流程

本插件的内部工作流程及核心逻辑如下：

1.  **定时轮询 (Scheduled Polling)**：
    *   插件内部运行异步轮询任务。
    *   遍历配置中所有已启用的 `目标会话 (Session)`，检查距离上次活跃或触发的时间是否达到了设定的间隔（含波动值）。
2.  **概率判定 (Probability Check)**：
    *   针对每个会话生成一个随机数，与当前该会话的“阈值概率”进行比对。
    *   **未命中**：跳过本次判定，并将该会话的下次触发概率按设定步长增加。
    *   **命中**：进入下一步 AI 决策流程。
3.  **上下文与记忆组装 (Context & Memory Assembly)**：
    *   获取目标会话的最近聊天记录，并获取当前北京时间。
    *   如果启用了记忆联动，会使用配置的关键词向 LivingMemory 插件请求检索相关记忆。
4.  **AI 智能决策 (AI Decision)**：
    *   将组装好的系统提示词（包含历史记录、时间、记忆、决策要求）发送给配置的 **决策模型**。
    *   模型根据提示词要求输出结果（例如输出 `YES` 或 `NO`）。
5.  **动作执行 (Execution)**：
    *   **决策为 NO（不发送）**：视为放弃本次主动聊天，增加下次命中概率。
    *   **决策为 YES（发送）**：将用户配置的“触发提示词”结合记忆和上下文，请求日常对话模型生成最终的主动回复内容，并发送给用户。发送成功后，重置该会话的触发概率为初始值。

## 使用方法

### 1. 指令说明

插件提供了以下指令供管理员或用户在聊天界面直接交互：

*   `/try_chat_status`
    *   **功能**：查看当前所有开启自动聊天会话的状态、当前概率、上次判定时间及最后活跃时间。
*   `/try_chat_force`
    *   **功能**：强制对当前所在的会话进行一次主动聊天判定与触发。
    *   **使用场景**：用于测试配置是否正确。注意，此指令依然会经过“AI 智能决策”步骤，如果 AI 认为当前不适合聊天，依然可能不会发送消息。

### 2. 提示词变量使用

在 WebUI 配置面板中填写“决策提示词”和“触发提示词”时，可使用以下变量：
*   `{{current time}}`：该变量会在每次检测时被自动替换为当前的北京时间（例如：`2026年5月10日 16:30:00`）。

## 配置说明

在 AstrBot 的 WebUI 管理面板中，找到 `astrbot_plugin_try_chat` 插件的设置项，可进行以下配置：

| 配置项 (Schema Key) | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `enabled_sessions` | `list` | `[]` | 启用的会话列表。格式为 `平台标识:消息类型:用户ID`。 |
| `base_probability` | `float` | `0.05` | 基础触发概率（0~1之间）。例如 0.05 代表初始 5% 的命中率。用户发消息后也会重置为此值。 |
| `probability_step` | `float` | `0.02` | 概率递增步长。每次未命中或 AI 拒绝发送时，下次概率将增加此数值。 |
| `polling_interval` | `int` | `1800` | 轮询间隔时间（秒）。默认为 1800 秒（半小时）。 |
| `interval_fluctuation`| `int` | `0` | 间隔时间随机波动范围（秒）。例如设置 300，则实际间隔在 1500~2100 秒之间随机。 |
| `history_len` | `int` | `10` | 判定和生成回复时，传递给模型的最近历史消息条数。 |
| `decision_model_id` | `string` | `""` | 用于执行“是否聊天”决策的大模型 ID。留空则默认使用 AstrBot 全局默认模型。 |
| `decision_prompt` | `text` | 见默认配置 | 决策提示词。**必须包含**判断标准，支持 `{{current time}}` 变量。要求模型输出 YES 或 NO。 |
| `trigger_prompt` | `text` | 见默认配置 | 当决策模型判定需要发消息时，用于触发日常聊天模型生成最终回复的系统指令。 |
| `decision_memory_keyword`| `string`| `"最近的约定"` | 在判断是否适合聊天时，调用 LivingMemory 记忆插件使用的固定搜索关键词。 |
| `max_retries` | `int` | `3` | 当大模型接口返回 503 或 504 等异常时，自动重试的最大次数。 |

## 注意事项

1.  **时区设置**：插件内部默认获取的时间为北京时间（Asia/Shanghai），请确保宿主机时间准确。
2.  **隐私保护声明**：本插件读取的“最近聊天记录”仅用于构建发给大模型的上下文提示词，绝不会进行未经授权的云端上传。
3.  **模型能力要求**：由于“决策”步骤需要模型具备良好的逻辑判断能力和指令遵循能力（如严格输出 YES/NO），建议 `decision_model_id` 配置为推理能力较强的模型。
